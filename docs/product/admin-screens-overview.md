# Admin Screens Overview

| Area | Screens | Officer scope | Super Admin scope |
| --- | --- | --- | --- |
| Auth | Admin Login | Can log in if officer | Can log in |
| Dashboard | Admin Dashboard | Own chorągiew metrics/tasks | Global metrics/tasks |
| Organization | Chorągiew List, Chorągiew Detail | Own chorągiew view | Create/edit/archive all |
| Brothers | Brother List, Brother Detail | Own chorągiew only | All |
| Candidates | Candidate Request List/Detail, Candidate List/Detail | Own/unassigned scoped flow | All |
| Prayers | Prayer List, Prayer Editor | If permitted by role policy | All |
| Events | Event List, Event Editor | Own chorągiew events | All |
| Announcements | Announcement List, Announcement Editor | Own chorągiew audiences | All |
| Roadmap | Definition List/Editor, Request List/Detail | Review scoped submissions | Manage definitions and all submissions |
| Silent Prayer | Event List, Event Editor | Own chorągiew/private/public if permitted | All |
| Audit | Audit Log | Limited or hidden | Global |

Admin screens must be utilitarian, dense, and clear. They must avoid ERP expansion beyond V1 operations.
